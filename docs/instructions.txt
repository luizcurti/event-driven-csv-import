Project Specification

# Project

Name

event-driven-data-ingestion

Objective

Build a serverless platform for large CSV imports using an event-driven architecture on AWS.

The system should handle hundreds of thousands of rows and process them in a distributed and scalable way.

Architecture

Internet -> AWS WAF -> API Gateway -> Upload Lambda -> Amazon S3 -> EventBridge -> Step Functions -> Split Lambda -> SQS -> Worker Lambdas -> Aggregator Lambda -> DynamoDB -> CloudWatch

Functional Flow

1. The user uploads a CSV.
2. The API responds immediately.
3. The file is stored in S3.
4. EventBridge receives an ObjectCreated event.
5. Step Functions starts.
6. The Split Lambda divides the file into chunks.
7. Each chunk generates an SQS message.
8. Workers process chunks in parallel.
9. Each Worker stores partial results.
10. The Aggregator checks when all chunks are done.
11. It consolidates the result.
12. It updates the import status.
13. CloudWatch records metrics and logs.
